Knowledge & Files

These nodes help your agent remember information, search libraries, and work with files on your computer.

1. Memory Node (The Notepad)

Saves or retrieves data from a local scratchpad so nodes can "talk" to each other.

Parameters:

operation get (read), set (save), or append (add to list).
key The name of the label for your info. (Example: user_name)
value The data to save (if using 'set'). Leave empty to use the previous node's result.
persistent True/False. If true, the data is saved even if you close the app.
Example:
Operation: "set"
Key: "age"
Value: "25"
Expected Result: You can now use {age} in any later LLM prompt to refer to "25".

2. Vector Memory (The Digital Library)

Searches through massive amounts of text based on "meaning" to find the most relevant context.

Parameters:

operation add (save text) or query (search for text).
collection Name of the "book" or collection. (Example: legal_docs)
text The text to save or the question to search for.
top_k How many matching results to return. (Example: 3)
Example:
Operation: "query"
Text: "What is the refund policy?"
Expected Result: Returns the specific paragraph from your uploaded documents that discusses refunds.

3. Variable Extractor (The Data Filter)

Picks out specific clean values from a messy AI response and saves them to memory.

Parameters:

mapping Pairs of "Old Label" : "New Label". (Example: {"name": "user_name"})
Example:
Mapping: {"city": "dest"}
Input JSON: {"city": "Tokyo", "reason": "Tourism"}
Expected Result: The word "Tokyo" is saved in memory under the label dest.

4. SQLite Node (The Database)

Runs professional database commands to store or find structured data.

Parameters:

db_path Path to the database file. (Example: orders.db)
query The SQL command. (Example: SELECT price FROM products WHERE id=1)
Example:
Query: "INSERT INTO logs (msg) VALUES ('Task Done')"
Expected Result: A new row is added to your local database table.
