Data & Memory Nodes

Memory Node

Sets or gets values from the workflow's shared memory. Use persistent=True to save data to disk between runs.

Variable Extractor

Extracts specific fields from a previous node's output (usually LLM JSON) and saves them into memory variables.

Vector Memory (RAG)

Store and search technical content using semantic embeddings. Useful for giving agents "long-term" knowledge or access to documentation.

SQLite Node

Run SQL queries against a local database. Use for structured data persistence.
