Organization & Files

These tools help you manage files and keep your workspace tidy.

1. File Reader

Reads text from a file on your computer.

Parameters:

path Location of the file. (Example: C:/Users/Bob/note.txt)
Example:
Path: "my_notes.txt"
Expected Result: The text written inside that file is sent to the next node.

2. PDF Reader

Specialized node that reads PDF documents or whole folders of PDFs.

Parameters:

path Path to the PDF file or a folder.
page_range Which pages to read. (Example: 1-3 or 5,7)
Example:
Path: "invoice.pdf"
Expected Result: Returns a list of text chunks, one for each page.

3. File Maker (Write)

Saves text or data into a new file on your computer.

Parameters:

path Where to save the file. (Example: results.txt)
mode w (write new) or a (add to end).
content Optional: Manually provide text to write. Otherwise, it writes the result of the previous node.
Example:
Path: "log.txt"
Mode: "a"
Input Result: "Agent completed task."
Expected Result: A new line is added to the "log.txt" file.

4. Inspect Node (Debug)

A "magnifying glass" for your workflow. It stops to show you what data is currently moving through the line.

No Parameters Needed

Example:
Position: Place it between an LLM and a File node.
Expected Result: You can see exactly what text the LLM generated before it gets saved to the file.
