Intelligence & Logic

These nodes act like the "Brain" of your agent. They help it think, make choices, and repeat tasks until a goal is met.

1. LLM Node (The Brain)

This is where the AI thinking happens. You give it instructions (a prompt), and it generates a text response.

Parameters:

api_base The address of your AI server. (Example: http://localhost:1234/v1)
model The name of the AI model to use. (Example: llama-3.2)
system_prompt Global instructions for the AI. (Example: You are a professional editor.)
user_prompt The specific request. Use {input} to include the previous node's result. (Example: Summarize: {input})
temperature Creativity level from 0 (strict) to 1 (creative). (Example: 0.7)
use_history True/False. If true, the AI remembers previous messages in the conversation.
Example:
User Prompt: "Translate to Spanish: {input}"
Input Value: "Hello, how are you?"
Expected Result: "Hola, ¿cómo estás?"

2. If/Else Node (The Choice Maker)

This node checks a condition and decides which path (True or False) to take.

Parameters:

condition A Python-style logic check. (Example: int({input}) > 80)
value Optional: Manually provide a value to check instead of using the previous result.
Example:
Condition: "len({input}) > 100"
Input Value: a long 200-character paragraph.
Expected Result: Execution continues along the True path.

3. While Loop (The Persistence Engine)

Repeats a task over and over as long as a condition remains true.

Parameters:

condition The loop continues while this is true. (Example: quality_score < 8)
max_iterations Safety stop. It will quit after this many tries. (Example: 10)
Example:
Condition: "check_finished == 'no'"
Expected Result: The workflow repeats its steps until another node changes check_finished to 'yes'.

4. JSON Dispatcher (The Tool Selector)

Automatically triggers different tools based on a specific word in the AI's response.

Parameters:

routing_key The name of the label to look for. (Example: action)
action_1_value If the label matches this word, go to 'Action 1'. (Example: search)
Example:
Routing Key: "command"
Action 1 Value: "email"
Input JSON: {"command": "email", "to": "bob"}
Expected Result: The workflow triggers the output connected to action_1.

5. Sequential Batch (The Step-by-Step List)

Takes a list of items and processes them one by one in order.

Parameters:

items_key The name of the list in memory. (Example: todo_list)
current_item_var The name to give the current item being processed. (Example: task)
Example:
Items Key: "cities"
Memory: {"cities": ["Madrid", "Paris", "Berlin"]}
Expected Result: The workflow runs 3 times. The first time, {task} will be "Madrid".
